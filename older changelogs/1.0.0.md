# BLOA 1.0.0-RC1

## Highlights

- Release candidate `1.0.0-RC1`
- `.baar` archive support with native package execution and inspection
- `new ClassName()` object instantiation for PHP/Java-like class creation
- Semicolon statement terminators supported for readable, PHP-style code
- cURL builtins for HTTP: `curl_get`, `curl_post`, `curl_request`
- SQLite builtins for local database access: `sqlite_query`, `sqlite_exec`

## Feature list

- Core language features: variables, arithmetic, string interpolation, comparisons, boolean logic, expressions
- Control flow: `if`/`else`, `while`, `for`/`foreach`, `repeat`, `break`, `continue`, `try`/`except`
- Functions and methods with argument handling and return values
- Classes, inheritance, and object creation using `new`
- Module loading with `use` and `require`
- Standard library utilities: `print`, `say`, `ask`, `range`, `len`, `str`, `int`, `float`, `append`, `math`, `io`, `string`, file and directory helpers
- BAAR archive helpers: `baar_create`, `baar_extract`, `baar_list`, `baar_read`
- `.baar` execution semantics: resolves `main.bloa`, `index.bloa`, or first `.bloa` entry when run directly
- Builtin helpers for `isset` and `unset`
- Added compatibility for semicolon-terminated statements outside of string literals

## Notes

- This release candidate captures the current BLOA feature set and prepares the runtime for wider package-based workflows.
- Fixed GitHub Actions release workflow to install Linux build dependencies for cURL and SQLite.
- Termux packages now use Android NDK cross-compilation for proper arm and arm64 Android binaries instead of packaging native Linux executables.
- Existing `.baar` archives can be used as deployable BLOA applications with a single execution command.
- The changelog is stored in `older changelogs/1.0.0.md` for reference while the RC1 candidate is finalized.
